# GIF-туториалы для AI Студия Че

Список коротких screen-cast'ов которые сделают онбординг понятным
без необходимости читать длинную доку. Каждый GIF длится **15–40 секунд**,
весит ≤ 3 МБ (silent loop, без звука), формат **animated WebP** или **MP4**
(MP4 предпочтительнее — меньше вес при той же чёткости, поддерживается тегом `<video autoplay loop muted playsinline>`).

Где разместить готовые файлы: `views/static/tutorials/<slug>.mp4` —
после этого I will ссылаться на них из подсказок UI.

## Приоритет 1 — обязательные (без них юзер теряется)

### 1. `bot-from-template.mp4` — «Создаём бота из шаблона за 60 секунд»
**Длина:** ~50 сек.
**Сценарий:**
1. Главная aiche.ru → клик «Чат-боты» в сайдбаре (1 сек).
2. На пустой странице — empty-state hero. Курсор останавливается на карточке «6 готовых шаблонов» (1 сек).
3. Клик «Из шаблона» → открывается галерея 6 шаблонов (1 сек).
4. Курсор наведён на «Запись на услугу» → клик (1 сек).
5. Открывается template setup — печатается название бота, услуги (3-4 строки), вставляется TG-токен, MAX-токен пропускается (15 сек).
6. Клик «Создать бота» → loader → success → карточка бота появляется в списке (3 сек).
7. На карточке зелёная точка «активен» + чип TG с галочкой (2 сек).
8. Финальный кадр: подсказка-стрелка к боту с подписью «Готово! Напишите ему в TG».

**Что показать главное:** простоту flow + что webhook регистрируется автоматически.

5. Появляется готовый бот в списке + всплывающее окно с graph-превью workflow (5 сек).
6. Финальный кадр: «Бот собран. Подключите токен и запустите».

### 3. `connect-tg-token.mp4` — «Получаем токен у @BotFather и подключаем»
**Длина:** ~30 сек.
**Сценарий:**
1. Открыт TG (можно Web). Чат с @BotFather.
2. Команда `/newbot` → ответ → имя бота → username → бот выдаёт токен.
3. Курсор копирует токен.
4. Переход на aiche.ru → клик карандаш на карточке бота → редактирование.
5. Вставка токена в поле «TG-токен» → Сохранить → Запуск.
6. Появляется зелёный чип «TG ✓».

**Что показать главное:** подключение реально занимает 30 сек.

### 4. `lead-magnet-upload.mp4` — «Загружаем PDF-лидмагнит и встраиваем в бота»
**Длина:** ~35 сек.
**Сценарий:**
1. На карточке бота — клик новой кнопки «Файлы» (иконка изображения).
2. Открывается модалка «Лидмагниты бота».
3. Drag-and-drop из проводника файла `чек-лист-seo.pdf` в drop-zone.
4. Прогресс «Загрузка…» → «✓ Загружен» (~3 сек).
5. Под файлом появляется поле с public-URL + кнопка «Копировать».
6. Клик «Копировать» → курсор переходит в воркфлоу-конструктор.
7. Открывается нода `output_tg_file` → вставка URL в поле `file_path`.
8. Клик Сохранить → готово.

**Что показать главное:** связку storage ↔ воркфлоу через копирование URL.

## Технические требования

- **Разрешение:** 1280×720 минимум (Retina-friendly если можешь 1920×1080).
- **Frame rate:** 24-30 fps.
- **Кодек:** H.264 baseline для MP4 (браузерная совместимость).
- **Без звука:** silent — `<video muted>`.
- **Цикл:** последний кадр ↔ первый кадр должны плавно сшиваться (если `loop`).
- **Курсор:** обязательно видим (часто скрывают, но это критично для туториалов).
- **Кадр-в-кадр текст-инструкция:** не нужна (UI говорит сам за себя),
  но для длинных пауз (>3 сек) можно добавить subtitle типа «AI собирает workflow…».

## Где разместить в UI

После того как пришлёшь GIF/MP4-файлы (положи в `views/static/tutorials/`),
я могу:
1. Добавить кнопку «📺 Как это работает» рядом с каждым CTA — открывает lightbox с туториалом.
2. На пустом списке ботов / сайтов — показывать первый туториал автоплеем.
3. В template-setup модалке — превью «как будет выглядеть» с туториалом.
4. На странице кабинета `/files` — туториал #4 (lead-magnet) при пустом списке.

Скажи когда сделаешь хоть один — встрою в UI и покажу как.
